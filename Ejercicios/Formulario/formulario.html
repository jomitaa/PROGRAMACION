Todos los formulario en la web manejan metodos para el envio de las diferentes peticiones que realiza el usuario. Debemos recordar que las mayotias de las aplicaciones en web trabajan sobre una arquitectura cliente servidor. Significa que el cliente realiza una peticion (request) y el servidor debe de atender mediante una respuesta (response) los metodos mas comunes son get:

Este metodo se encara de enviar la informacion del form por medio de sÂ¿la salida o url estandar del navegador, esto propicia que pueda existir inyecciones de codigo realizando violaciones a la seguridad del sistema.

Post:

Este metodo se encarga de enviar la informacion igual por el metodo estandar; pero mediante un ocultamiento, esto significa que no es visible para el usuario. Lo que nos ofrece un mayor grado de seguridad en el manejo en el sistema

Get solo se usa para operaciones de lectura Post se usa para cambiar la informacion de un recurso por parte del servidor. (lo mas comun de post es insertar un Registro) Delete sirve para eliminar un recurso en especifico. Put almacena la informacion de un recurso particular, para su actualizacion. Head Regresa el codigo de respuesta y cualquier cabecera HTTP asociada.
